üé® Visual Identity
Core Design Philosophy
Modern glassmorphism with gradient accents, creating depth through transparency and blur effects while maintaining excellent readability in a dark theme environment.
üé® Color System
Primary Palette
cssBackground Layers:
- Base: bg-gradient-to-br from-gray-900 via-gray-800 to-gray-900
- Cards: bg-gray-800/30 or bg-gray-800/50 (with backdrop-blur-sm)
- Overlays: bg-black/60 (with backdrop-blur-sm)
- Elevated: bg-gray-900/50 or bg-gray-900/70

Gradients:
- Primary: from-blue-600 to-purple-600 (CTAs, headers)
- Alternate: from-blue-500 to-purple-500 (avatars, progress)
- Hover: from-blue-700 to-purple-700

Text Hierarchy:
- Primary: text-white
- Secondary: text-gray-300
- Muted: text-gray-400
- Disabled: text-gray-500

Status Colors:
- Success: bg-green-500/10 with text-green-400
- Warning: bg-yellow-500/10 with text-yellow-400
- Error: bg-red-500/10 with text-red-400
- Info: bg-blue-500/10 with text-blue-400

üîë Key Design Rules

Always use glassmorphism for cards: bg-gray-800/30 backdrop-blur-sm
Gradient accents for important actions and headers
Icons with labels for better UX
Smooth transitions (200-300ms) on all interactive elements
Consistent border styling: border-gray-700/50
Hover states should provide clear feedback
Use emojis in select options for visual appeal
Maintain spacing hierarchy: 8 > 6 > 4 > 2
Profile elements use gradient backgrounds
Error states use red theme consistently

üì± Responsive Considerations

Mobile first approach
Cards stack on mobile, grid on desktop
Sidebar collapses on mobile
Modals take full screen on mobile
Touch-friendly button sizes (min 44px)

This design system ensures visual consistency and modern aesthetics across all components while maintaining excellent usability and accessibility.
